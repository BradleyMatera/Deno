Project Structure

This project is organized into a clean and modular structure that separates concerns, making it easy to maintain and scale. Below is an overview of the structure:


.
├── public/          # Contains static assets like HTML and CSS files
├── models/          # Database models (e.g., User.js for MongoDB)
├── main.js          # Main server file, starts the Deno server
├── deps.js          # Dependency imports (e.g., Express, MongoDB, Dex ORM)
├── test.js          # Contains test cases for API endpoints
├── deno.json        # Task runner configuration
├── deno.lock        # Lock file for dependencies
├── import_map.json  # Import map for module resolution

Task Runner

Deno's built-in task runner makes it easy to manage development workflows. Here's the configuration for the tasks:


{
  "tasks": {
    "start": "deno run --allow-net --allow-read main.js",
    "dev": "deno run --watch --allow-net --allow-read main.js",
    "test": "deno test --allow-net"
  }
}

With these tasks, you can easily start the server in production, run it in development mode with file watching, and run tests to ensure everything is working properly.
