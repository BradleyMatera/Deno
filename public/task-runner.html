Deno's Task Runner

In this project, we use Deno's built-in task runner to manage development workflows. It allows us to easily run commands like starting the server, watching for file changes, and running tests.

Task Configuration


{
  "tasks": {
    "start": "deno run --allow-net --allow-read main.js",
    "dev": "deno run --watch --allow-net --allow-read main.js",
    "test": "deno test --allow-net"
  }
}

Project Structure


.
├── public/          # Static files like HTML, CSS
├── models/          # Database models
├── main.js          # Entry point of the server
├── test.js          # Tests
├── deps.js          # Dependencies (e.g., Express, MongoDB)
├── deno.json        # Task runner configuration
├── import_map.json  # Module resolution for dependencies
